# Context Service Contracts

This document records stable contracts for the standalone event-sourced Context Service. It covers event storage, trace projection, agent context, redaction, evaluation, and benchmark inputs/outputs.

## Event Contract

`ContextEvent` is the append-only event shape managed by `EventStore`.

| Field | Meaning |
| --- | --- |
| `event_id` | Unique event id generated by EventStore unless provided. |
| `task_id` | Task id and task-local storage key. |
| `trace_id` | Optional trace correlation id. |
| `span_id` | Optional span id. |
| `parent_span_id` | Optional parent span id. |
| `run_id` | Optional run or generation id. |
| `seq` | Monotonic task-local sequence assigned by EventStore. |
| `type` | Event type, for example `TRACE_NODE_APPENDED` or a supported domain event. |
| `category` | Optional event category. |
| `payload` | Event payload. Must avoid secrets and large unsafe raw fields. |
| `created_at` | Event creation timestamp assigned by EventStore unless provided. |
| `schema_version` | Event schema version. |
| `producer` | Component that produced the event. |
| `idempotency_key` | Optional idempotency key for duplicate append handling. |

Read behavior:

- `readEvents(taskId)` returns raw events.
- `readSafeEvents(taskId)` returns events after redaction overlay.
- `readEventsByType(taskId, type)` filters raw events by event type.
- `getLatestEventSeq(taskId)` returns the latest task-local sequence.

Write behavior:

- `appendEvent(taskId, event, options?)` appends only.
- `expectedSeq` guards optimistic concurrency.
- Idempotency is based on `idempotency_key` and normalized event content.
- Existing events are not rewritten.

## Trace Contract

`TraceNode` is a projected task artifact, decision, or context node.

Common fields:

- `id`
- `type`
- `status`
- `summary`
- `metadata`
- `source_event_id`
- `created_at`
- `updated_at`

`TraceEdge` connects two trace nodes.

Common fields:

- `id`
- `from_node_id`
- `to_node_id`
- `relation`
- `source_event_id`
- `created_at`

Dependency direction is fixed:

```text
A depends_on B = from_node_id A -> to_node_id B
```

That means a child node points to the parent dependency it relies on.

`TraceView` contains:

- `projector_version`
- `task_id`
- `source_event_count`
- `source_last_seq`
- `events_hash`
- `view_hash`
- `generated_at`
- `metadata`
- `nodes`
- `edges`

Projection result shape:

```js
{
  trace_view,
  projection_report
}
```

Projection rules:

- Projection reads safe events when redaction manifests exist.
- Projection records errors/warnings in `projection_report`.
- Invalid events must not pollute `trace_view`.
- Projection must not write EventStore.
- `view_hash` excludes volatile fields such as `generated_at`.

## AgentContext Contract

`AgentContext` is the filtered, budgeted object produced for downstream agent use.

Expected fields include:

- `task_id`
- `agent_name`
- `current_node_id`
- `source_node_ids`
- `source_event_ids`
- `context`
- `budget_report`
- `privacy_report`
- `generated_at`

`CONTEXT_BUILT` is the current Context Service event emitted by `AgentContextBuilder`.
`AGENT_CONTEXT_BUILT` is a legacy-compatible projector alias.

Allowed context-built metadata:

- lightweight task/context metadata
- source node ids
- source event ids
- budget report
- privacy report
- context hash or cache reference

Forbidden in agent-facing context and context build metadata:

- `full_context`
- raw sandbox log
- full patch diff
- full chat history
- unfiltered tool result payloads
- secrets or private credentials

Budget and privacy rules:

- `ContextBudgetManager` removes forbidden raw fields and truncates oversized fields.
- `PrivacyFilter` redacts sensitive keys and text patterns.
- `context_cache` must store filtered context only.

## Redaction Contract

`RedactionManifest` shape:

```js
{
  manifest_id,
  task_id,
  affected_event_ids,
  redacted_paths,
  reason,
  created_at
}
```

Allowed `reason` values:

- `secret_leak`
- `pii_leak`
- `user_request`
- `policy`

`applyRedactionOverlay(events, manifests)`:

- deep clones input events
- matches events by `affected_event_ids`
- replaces configured dot paths with `[REDACTED]`
- safely skips missing paths
- does not modify raw event objects
- does not rewrite event files

`readSafeEvents(taskId)`:

- reads raw events
- reads redaction manifests
- applies overlay
- returns safe event copies

Redaction is incident recovery. It does not replace proactive privacy filtering before context exposure.

## ContextEval / Benchmark Contract

`ContextEvalRunner` evaluates an existing or built `AgentContext`.

Primary eval inputs:

- `evalCase.task_id`
- `evalCase.target_agent`
- `evalCase.current_node_id`
- `evalCase.context`
- `evalCase.expected_source_nodes`
- `evalCase.forbidden_source_nodes`
- `evalCase.expected_constraints`
- `evalCase.expected_attributions`

Primary metrics:

- `dependency_recall`
- `noise_rate`
- `constraint_recall`
- `source_attribution_accuracy`
- `privacy_leakage`
- `replay_accuracy`

Quality report dimensions:

- `relevance`
- `sufficiency`
- `provenance`
- `economy`
- `isolation`
- `overall_score`

`ContextBenchmark` compares context strategies with AgentContext-like inputs.

Supported baseline strategies:

- `recent_messages`
- `global_summary`
- `dependency_chain`

Benchmark output includes:

- per-strategy metrics
- per-strategy quality report
- `context_size_chars`
- winner selection by `overall_score`, then smaller context size, then strategy order

Benchmark does not build real contexts automatically and does not mutate EventStore.
